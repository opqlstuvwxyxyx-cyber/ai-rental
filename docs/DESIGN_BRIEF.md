## 3. デザイン方針

### トーン & マナー

- **高級感のあるダークUI**（黒〜ダークグレー基調）
- アクセントカラー: **ゴールド**（`#c9a962` 系）
- テキスト: クリーム（`#f5f0e8`）+ ミュートグレー
- フォント: Playfair Display（見出し）+ Noto Sans JP（本文）

### 目指す印象

- Netflix / 高級ブランドのライブラリ感
- 「デモサイト」ではなく「本番サービス」に見えること
- 余白を活かした、情報が整理されたレイアウト

### NG（やってはいけない）

- [ ] 「ローカル確認用」「Local Preview」などデモ感の文言
- [ ] `alert()` によるエラー表示
- [ ] 英語と日本語の無秩序な混在（ラベルは基本日本語）
- [ ] 機能しないボタン（DL・申し込みなど）
- [ ] 過度なアニメーション・派手すぎるグラデーション
- [ ] スマホでログイン・設定に到達できないUI

## 6. ページ別要件

### `/materials` 素材一覧

- [ ] 素材件数が一目で分かる
- [ ] カテゴリフィルタが直感的
- [ ] カードにホバーで再生アイコン or プレビュー
- [ ] 0件時の空状態メッセージ
- [ ] グリッドがモバイル2列・デスクトップ4〜5列

### `/sozai/[id]` 素材詳細

- [ ] 動画プレビューが大きく見やすい
- [ ] レンタル / 返却 / DL の導線が明確
- [ ] 枠がいっぱいのとき親切なエラー
- [ ] 関連素材の表示

### `/` トップ

- [ ] サービス価値が3秒で伝わる
- [ ] CTA（素材を探す / プランを見る）が目立つ
- [ ] カテゴリ・おすすめ素材への導線

### `/plan` 料金

- [ ] 3プランの比較が一目で分かる
- [ ] 申し込み → 登録でプランが引き継がれる
- [ ] FAQ または信頼要素

### `/settings` 設定

- [ ] プラン・枠の使用状況が視覚的
- [ ] フォームにラベルあり
- [ ] デモアカウントと本番ユーザーの区別

## 9. 作業ルール（Claude Code × Cursor 連携）

1. **1ラウンド = 1テーマ**（例: 素材一覧だけ）
2. 同時に同じファイルを触らない → ブランチを分ける
3. 変更前にこのブリーフと `REVIEW_LOG.md` を読む
4. 作業後は `REVIEW_LOG.md` に結果を追記
5. コミットメッセージは `[UI]` `[Fix]` `[Docs]` など接頭辞を付ける
6. push 前に `npm run build` が通ることを確認

### ブランチ命名

```
claude/ui-round-1-materials
cursor/ui-round-1-polish
```
